# API設計

## エンドポイント一覧

| ID | 機能 | エンドポイント（URI） | HTTPメソッド | 認証 | 備考 |
|:---:|------|-----------------------|:------------:|:---:|------|
| AP01 | 書籍一覧 | `/api/v1/books` | GET | 不要 | 検索・絞り込み・ページネーション対応 |
| AP02 | 書籍詳細 | `/api/v1/books/{id}` | GET | 不要 | レビュー一覧、平均評価、ジャンル情報を含む |
| AP03 | 書籍登録 | `/api/v1/books` | POST | 不要 | JSON形式で書籍を登録。バリデーションを実施 |
| AP04 | 書籍更新 | `/api/v1/books/{id}` | PUT | 不要 | 既存書籍を更新（自身のISBNは重複チェック対象外） |
| AP05 | 書籍削除 | `/api/v1/books/{id}` | DELETE | 不要 | 関連するレビューなどをカスケード削除 |

---

## 使用するHTTPステータスコード

| ステータスコード | 説明 | 利用場面 |
|------------------|------|----------|
| **200 OK** | リクエスト成功 | データ取得・更新処理が正常に完了した場合 |
| **201 Created** | リソース作成成功 | 新規書籍登録が正常に完了した場合 |
| **204 No Content** | リソース削除成功 | 削除処理が正常に完了し、レスポンスボディを返さない場合 |
| **400 Bad Request** | 不正なリクエスト | リクエスト形式やパラメータが不正な場合 |
| **404 Not Found** | リソースが存在しない | 指定された書籍IDが存在しない場合 |
| **422 Unprocessable Entity** | バリデーションエラー | 入力値がバリデーションルールを満たさない場合 |

---

## 備考

- APIのレスポンス形式は **JSON** とする。
- エラーレスポンスは、HTTPステータスコードに加えてエラー内容をJSON形式で返却する。
- APIのバージョンは URI に `/v1` を付与して管理する。
- 書籍登録・更新時は `FormRequest` を利用してバリデーションを実施する。
- 書籍削除時は、関連するレビューや中間テーブルのデータを外部キー制約（`ON DELETE CASCADE`）により削除する。
