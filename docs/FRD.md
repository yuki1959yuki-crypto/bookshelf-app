| 画面名 | 操作 | URL / メソッド | 完了条件 |
|--------|------|----------------|----------|
| 書籍登録画面 | ページを表示する | `GET /books/create` | 認証時→タイトル・著者・ISBN・出版日・説明・画像URLの入力欄と、全ジャンル一覧のチェックボックス（複数選択可）が表示される（応用時は「ISBN書籍情報自動入力フォーム」も含む） |
|  | 全項目を入力して「登録」ボタンを押す | `POST /books` | バリデーション通過時→書籍が登録され、ジャンル紐付けが作成される<br>失敗時→バリデーションエラーが表示される |
| 書籍編集画面 | ページを表示する | `GET /books/{book}/edit` | 認証＋作成者本人→既存データが初期値として表示された編集フォームが表示される<br>認可が必要 |
|  | 内容を変更して「更新」ボタンを押す | `PUT /books/{book}` | 認証＋作成者本人＋バリデーション通過→書籍とジャンル紐付けが更新される<br>認可が必要 |
| レビュー編集画面 | ページを表示する | `GET /reviews/{review}/edit` | 認証＋投稿者本人→既存のrating/commentが初期値として表示された編集フォームが表示される<br>認可が必要 |
|  | 内容を変更して「更新」ボタンを押す | `PUT /reviews/{review}` | 認証＋投稿者本人＋バリデーション通過→レビューが更新される<br>認可が必要 |
| お気に入り一覧画面 | ページを表示する | `GET /favorites` | 認証時→ログインユーザーのお気に入り書籍が10件/ページでページネーション表示される |
|  | 書籍タイトルを押す | `GET /books/{book}` | 書籍詳細画面に遷移する |
| ジャンル一覧画面 | ページを表示する | `GET /genres` | 認証時→各ジャンルに紐づく書籍数付きでジャンル一覧が表示される |
|  | ジャンル名を押す | `GET /genres/{genre}` | ジャンル詳細画面に遷移する |
|  | 「ジャンルを登録」ボタンを押す | `GET /genres/create` | ジャンル登録画面に遷移する |
|  | 各ジャンルの「編集」ボタンを押す | `GET /genres/{genre}/edit` | ジャンル編集画面に遷移する |
|  | 各ジャンルの「削除」ボタンを押す | `DELETE /genres/{genre}` | 書籍紐付きの有無に応じて適切に処理されること<br>紐付きがある場合は削除を制限すること |
| ジャンル詳細画面 | ページを表示する | `GET /genres/{genre}` | 認証時→ジャンル名と紐づく書籍がページネーション（10件/ページ）で表示される |
| ジャンル登録画面 | ページを表示する | `GET /genres/create` | 認証時→ジャンル名入力フォームが表示される |
|  | ジャンル名を入力して「登録」ボタンを押す | `POST /genres` | バリデーション通過時→ジャンルが登録される<br>失敗時→バリデーションエラーが表示される |
| ジャンル編集画面 | ページを表示する | `GET /genres/{genre}/edit` | 認証時→現在のジャンル名が初期値として表示された編集フォームが表示される |
|  | ジャンル名を変更して「更新」ボタンを押す | `PUT /genres/{genre}` | バリデーション通過時→ジャンル名が更新される<br>失敗時→バリデーションエラーが表示される |
| ランキング画面 | ページを表示する | `GET /ranking` | レビュー平均評価のTOP10書籍が降順で表示される<br>レビューがない書籍は表示されない<br>ゲストもアクセス可能 |
| ★ マイ読書レポート画面 | ★ ページを表示する | `GET /reports` | 認証時→以下の4種の統計情報が表示される<br>1. 基本サマリー: 総レビュー数、読了冊数（ユニーク書籍数）、平均評価点<br>2. 評価分布: 1〜5星ごとの件数を横バーで表示<br>3. 高評価書籍TOP5: 4星以上の書籍を評価の高い順に最大5件表示（書籍詳細リンク付き）<br>4. ジャンル別評価傾向TOP5: ジャンルごとの平均評価と件数を高い順に最大5件表示（ジャンル詳細リンク付き） |
| ログアウト | ヘッダーの「ログアウト」ボタンを押す | `POST /logout` | セッションが破棄されログイン画面にリダイレクトされる |
| 公開API | 書籍一覧を取得する | `GET /api/v1/books` | JSON形式で書籍一覧が返される<br>検索・絞り込み・ページネーション対応。各書籍にジャンル情報・平均評価・レビュー件数を含めること |
|  | 書籍詳細を取得する | `GET /api/v1/books/{book}` | JSON形式で書籍詳細（ジャンル・レビュー含む）が返される<br>存在しないIDの場合はエラーレスポンスを返すこと |
|  | 書籍を新規登録する | `POST /api/v1/books` | バリデーション通過時→書籍が登録される<br>失敗時→エラーレスポンスを返す |
|  | 書籍を更新する | `PUT /api/v1/books/{book}` | バリデーション通過時→書籍が更新される<br>存在しないIDの場合はエラーレスポンスを返すこと |
|  | 書籍を削除する | `DELETE /api/v1/books/{book}` | 書籍が削除される<br>関連データも適切に処理される<br>存在しないIDの場合はエラーレスポンスを返すこと |
|  | ★ Sanctum認証を要求する書き込み系API | `POST/PUT/DELETE /api/v1/books`（応用段階） | 書き込み系エンドポイントにSanctumトークン認証を導入する<br>認証済み・書籍所有者のみが書き込み可能<br>未認証時・認可エラー時は適切なHTTPステータスを返すこと |

|  | ★ 「編集」ボタンを押す | `GET /reading-plans/{plan}/edit` | 認証＋所有者→読書計画編集画面に遷移。認可が必要 |
|  | ★ 「削除」ボタンを押す | `DELETE /reading-plans/{plan}` | 認証＋所有者→計画が削除される。認可が必要 |
|  | ★ 「新規計画作成」ボタンを押す | `GET /reading-plans/create` | 認証時→読書計画作成画面に遷移 |
| ★ 読書計画作成画面 | ★ ページを表示する | `GET /reading-plans/create` | 認証時→書籍プルダウンと期日入力フォームが表示される。|
|  | ★ 書籍と期日を入力して「登録」ボタンを押す | `POST /reading-plans` | バリデーション通過時→計画が登録される。失敗時→バリデーションエラーが表示される |
| ★ 読書計画編集画面 | ★ ページを表示する | `GET /reading-plans/{plan}/edit` | 認証＋所有者→既存の期日が初期値として表示された編集フォームが表示される。認可が必要 |
|  | ★ 期日を変更して「更新」ボタンを押す | `PUT /reading-plans/{plan}` | 認証＋所有者＋バリデーション通過→計画情報が更新される。認可が必要 |
