# 画面遷移図

---

# 1. 認証系フロー（Laravel Fortify）

## ログインページ（`/login`）

- 認証成功 → **書籍一覧ページ（`/books`）**へ遷移
- 「会員登録」リンク → **会員登録ページ（`/register`）**へ遷移

## 会員登録ページ（`/register`）

- 登録成功 → **書籍一覧ページ（`/books`）**へ遷移

---

# 2. 書籍管理系フロー

## 書籍一覧・検索画面（`/books`）

- 書籍名をクリック → **書籍詳細画面（`/books/{id}`）**へ遷移
- 「新規登録」ボタン → **書籍登録画面（`/books/create`）**へ遷移

## 書籍詳細画面（`/books/{id}`）

- 「編集」ボタン → **書籍編集画面（`/books/{id}/edit`）**へ遷移
- 「削除」ボタン → 削除実行後、**書籍一覧画面（`/books`）**へリダイレクト

## 書籍登録画面（`/books/create`）

- 「保存」ボタン → 登録成功後、**書籍一覧画面（`/books`）**へリダイレクト

## 書籍編集画面（`/books/{id}/edit`）

- 「更新」ボタン → 更新成功後、**書籍詳細画面（`/books/{id}`）**へリダイレクト




## エラーハンドリング

- バリデーションエラーが発生した場合は、入力内容を保持したまま元の画面（登録画面・編集画面）へ自動的にリダイレクトします。


## リダイレクト

- 保存・更新・削除などの操作完了後は、操作結果が分かりやすいように、必要に応じてフラッシュメッセージを表示する設計とします。
